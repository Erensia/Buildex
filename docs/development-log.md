## 2026-07-16 — 명조 MVP 범위 확정

- 첫 지원 게임은 **명조: 워더링 웨이브**로 결정했다. 젠레스 존 제로는 공식 앱의 지원 범위가 넓어, 첫 MVP의 차별화 대상에서는 제외한다.
- 콘텐츠 데이터는 공식 버전 공지·패치 노트와 게임 내 공개 수치를 기준으로 운영한다. 초기에는 관리자 수동 입력·검수 방식으로 갱신하며, 게임 클라이언트 분석·비공개 API 호출·계정 자동 연동·미출시 정보는 범위에서 제외한다.
- MVP는 실전 DPS나 자동 로테이션 계산기가 아니라, **스탯과 조건부 버프를 투명하게 합산하는 빌드·파티 비교 도구**로 정의했다.
- 캐릭터 평가는 `준종결`, `종결`, `극종결`의 세 기준으로 제공한다. 각 기준은 캐릭터·무기·파티·운용 조건별 목표 스탯과 필수 조건으로 관리한다.
- 초기 지원 대상은 주요 딜러·범용 서포터를 정확하게 지원한 뒤 확대하는 방식으로 정했다. 첫 검증 파티는 **장리–루파–브랜트(장루브)**, 두 번째 확장 파티는 **젠니–페비–벨리나**다.
- 첫 구현은 장루브 파티를 기준으로 한다. 장리의 기준 빌드는 **0공명 + 전용 무기**로 시작하고, 다른 무기 및 고공명 변형은 후속 지원한다.
- 사용자는 캐릭터·무기·에코를 수동으로 입력한다. 에코 세트와 주옵은 목록에서 선택하고, 에코 레벨은 만렙 고정으로 처리한다. 부옵은 최대 5개를 수치와 함께 입력한다.
- 에코 5개 묶음은 이름을 붙인 빌드 프리셋으로 저장·불러올 수 있게 한다. 결과에는 개인 최종 스탯, 파티 버프 적용 전·후 차이, 목표 등급 판정, 에코별 교체 우선순위를 표시한다.

## 2026-07-16 — 장루브 계산 기반 구현

- `src/lib/formula/build-calculator.ts`에 캐릭터·무기·에코의 기초 공격력, 공격력%, 고정 공격력을 합산하는 독립 계산기를 추가했다. 치명타, 공명 효율, 융해 피해 보너스, 공명 스킬 피해 보너스도 같은 결과 모델로 반환한다.
- 파티 버프는 자동으로 가정하지 않는다. UI가 버프 발동 조건을 확인한 뒤 활성 버프 ID를 넘길 때만 합산하므로, 결과의 적용 전제와 계산 근거를 표시할 수 있다.
- 장리 0공명 + 전용 무기 기준의 `준종결`·`종결`·`극종결` 목표 스탯을 데이터 규칙으로 분리했다. 규칙 평가기는 달성한 최고 등급과 다음 등급에 부족한 항목을 반환한다.
- 계산식과 버프 적용, 등급 판정의 단위 테스트를 추가했다. 실제 캐릭터·무기·에코 데이터 시드는 후속 검수 작업으로 분리했다.

## 2026-07-16 — 장루브 검수 스냅샷 시드

- 나무위키 검수 데이터 기준으로 명조, 장리·루파·브렌트와 각 전용 무기, 추천 4코 에코 3종, 에코 세트 3종, 5성 +25 주옵을 초기 시드로 추가했다.
- 명조 3.5 버전을 `dataVersion`에 기록하고, 모든 시드에 `sourceSnapshot: 2026-07-16`과 출처 URL을 함께 보관한다.
- 에코 세트와 에코의 다대다 연결, 코스트별 주옵을 위한 모델을 추가했다. 현재 세트별 추천 4코만 연결했으며, 전체 구성 에코는 후속 검수 데이터로 확장한다.
- 브렌트 반주 버프의 용융 피해 보너스를 20%로 정정했고, 장리 전무 최대 스택 효과를 사용자가 명시적으로 선택하는 조건부 버프로 추가했다.
- Docker PostgreSQL 16 컨테이너에서 초기 마이그레이션과 검수 스냅샷 시드를 실제 적용했다. 게임 버전 3.5, 캐릭터 3종, 무기 3종, 추천 에코 3종, 에코 세트 3종, 주옵 7종의 저장을 조회로 검증했다.

## 2026-07-16 — 장리 빌드 입력·결과 화면

## 2026-07-16 — 인증 보강 및 빌드 프리셋 저장

- `/build`를 보호 경로로 전환했다. 비로그인 사용자는 로그인 화면으로 이동하고, 인증 후에는 원래 요청한 `/build`로 복귀한다.
- 빌드 화면에 세션 사용자 표시와 로그아웃을 추가했다.
- `GET`·`POST /api/build-profiles`, `DELETE /api/build-profiles/:id`를 구현했다. 모든 요청에서 현재 세션 사용자를 확인하고, 삭제는 소유한 프리셋에만 허용한다.
- 빌드를 저장할 때 클라이언트 결과를 그대로 신뢰하지 않고 서버에서 주옵·부옵·활성 버프를 다시 계산해 결과 스냅샷으로 저장한다. 화면에서 저장 목록 조회와 삭제를 제공한다.
- ESLint, Vitest 단위 테스트, Next.js 프로덕션 빌드를 통과했다. Docker PostgreSQL 16에서 마이그레이션 2건과 확장 스키마, 명조 시드(게임 1·캐릭터 3·무기 3·에코 3·세트 3·주옵 7)를 조회로 검증했다. 인증이 필요한 저장 API의 종단 간 통합 테스트는 후속 작업이다.
