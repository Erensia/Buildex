# Buildex 개발 로그

## 2026-07-16 — 기반 구축

- Next.js, PostgreSQL, Drizzle ORM, Auth.js Credentials 기반을 구성했다.
- 사용자, 인증 식별자, 게임 데이터, 빌드 프로필 스키마와 마이그레이션을 추가했다.
- 창리 기준 스탯 계산기와 단위 테스트를 추가했다.

## 2026-07-16 — 빌드 저장

- 로그인 사용자의 빌드 목록 조회, 생성, 삭제 API를 구현했다.
- 서버에서 빌드 입력을 검증하고 계산 결과와 버전을 함께 저장하도록 했다.

## 2026-07-16 — 데이터 선택 및 편집

- `/api/build-data`로 DB 기반 캐릭터·무기·에코 세트·주옵션 선택 데이터를 제공한다.
- 캐릭터에 맞는 무기 타입만 선택하도록 빌드 UI를 변경했다.
- 저장 빌드를 불러오고 수정(PUT)·삭제할 수 있게 했다.
- 문서 네 개를 UTF-8 한국어 원문으로 정규화했다.

## 2026-07-16 — MVP 에코 선택지 보완

- 1·3 코스트 에코와 세트 연결 데이터를 추가해 4-3-3-1-1 빌드 슬롯을 모두 선택할 수 있게 했다.
- 저장 입력에서 5개 슬롯, 슬롯별 코스트, 에코 선택 여부를 검증하도록 강화했다.
- 에코 데이터는 출처 URL과 스냅샷을 함께 기록해 다음 갱신의 기준으로 사용한다.

## 2026-07-16 — 에코 세트 효과 계산

- 에코 슬롯마다 세트를 선택해 2세트/5세트 조합을 표현할 수 있게 했다.
- 2세트 효과는 자동 적용하고, 조건이 있는 5세트 효과는 사용자 확인 후 클라이언트와 서버 계산에 동일하게 반영한다.

## 2026-07-16 — 빌드 참조 무결성 검증

- 서버가 무기 타입, 에코 코스트, 에코-세트 소속, 주옵션 가능 여부를 DB 기준으로 검증하도록 했다.
- 화염 발톱 세트의 1·3 코스트 선택지와 연결 데이터를 보충했다.

## 2026-07-16 — 에코별 부옵션 및 조건부 버프

- 부옵션을 에코별로 입력·저장·복원하도록 변경했다.
- 창리 전용무기 버프는 해당 무기 장착 시에만 표시하고 서버 계산에도 반영한다.

## 2026-07-16 — 등급 기반 부옵션 추천

- 창리 빌드는 다음 등급에 도달하기 위해 부족한 스탯을 결과 패널에서 바로 확인할 수 있게 했다.

## 2026-07-16 — 로컬 환경 설정 안정화

- `.env.local`에 로컬 데이터베이스 연결 정보와 고정 `NEXTAUTH_SECRET`, `NEXTAUTH_URL`을 추가했다.
- JWT 세션 복호화 오류를 방지하기 위해 NextAuth secret을 개발 서버 재시작 시에도 유지하도록 했다.
- `drizzle.config.ts`가 `.env.local`을 자동으로 읽도록 `dotenv`를 추가해, 마이그레이션 시 별도의 환경변수 명령이 필요 없게 했다.
