# Buildex 개발 로그

## 2026-07-16 — 기반 구축

- Next.js, PostgreSQL, Drizzle ORM, Auth.js Credentials 기반을 구성했다.
- 사용자, 인증 식별자, 게임 데이터, 빌드 프로필 스키마와 마이그레이션을 추가했다.
- 창리 기준 스탯 계산기와 단위 테스트를 추가했다.

## 2026-07-16 — 빌드 저장

- 로그인 사용자의 빌드 목록 조회, 생성, 삭제 API를 구현했다.
- 서버에서 빌드 입력을 검증하고 계산 결과와 버전을 함께 저장하도록 했다.

## 2026-07-16 — 데이터 선택 및 편집

- `/api/build-data`로 DB 기반 캐릭터·무기·에코 세트·주옵션 선택 데이터를 제공한다.
- 캐릭터에 맞는 무기 타입만 선택하도록 빌드 UI를 변경했다.
- 저장 빌드를 불러오고 수정(PUT)·삭제할 수 있게 했다.
- 문서 네 개를 UTF-8 한국어 원문으로 정규화했다.

## 2026-07-16 — MVP 에코 선택지 보완

- 1·3 코스트 에코와 세트 연결 데이터를 추가해 4-3-3-1-1 빌드 슬롯을 모두 선택할 수 있게 했다.
- 저장 입력에서 5개 슬롯, 슬롯별 코스트, 에코 선택 여부를 검증하도록 강화했다.
- 에코 데이터는 출처 URL과 스냅샷을 함께 기록해 다음 갱신의 기준으로 사용한다.
