# Buildex 개발 로그

## 2026-07-17 — 플래너 데이터 표시와 부옵션 규칙 정정

- 게임 데이터 API 오류 응답이 플래너 상태에 들어와 화면이 중단되던 문제를 방어하고, 오류 메시지를 사용자에게 표시하도록 수정했다.
- 등급 달성·추천 개선 안내에서 공격력은 수치로, 치명타·공명 효율·피해 보너스는 `%` 단위로 표시하도록 통일했다.
- 에코 부옵션에서 속성 피해 보너스를 제거하고 일반 공격·강공격·공명 스킬·공명 해방 피해 보너스로 교체했다. 공격력%·HP% 및 유형별 피해 보너스는 동일한 실제 8단계 롤 범위(`6.4%`~`11.6%`)만 허용한다.

## 2026-07-17 — 문서와 구현 정합성 점검

- 등급 기준은 최초의 창리 전용 구현에서 자니·히유키까지 확장됐다. 요구사항과 기본 설계의 지원 범위를 이에 맞춰 정정했다.
- 데이터 릴리스 식별자는 저장하지만, 당시 릴리스로 빌드를 읽고 현재 릴리스로 명시적으로 복제·재계산하는 사용자 흐름은 아직 구현 대상임을 문서에 명시했다.

## 2026-07-17 — 빌드 재현성과 릴리스 안전성 보강

- 저장 빌드를 열 때 해당 빌드의 `data_release_id`로 공개·보존 릴리스 데이터를 읽고, 수정 API도 동일 릴리스를 기준으로 입력을 다시 검증하도록 변경했다. 최신 데이터로 옮기는 작업은 현재 공개 데이터로의 복제로 분리했다.
- 계산식 버전은 클라이언트 입력이 아닌 서버의 `FORMULA_VERSION`으로 기록하도록 통일하고, 현재 명조 3.5 계산식 번들로 갱신했다.
- 게임당 공개 릴리스는 하나만 허용하는 부분 고유 인덱스를 추가했다. Publish는 초안 상태를 조건으로 전환하고 충돌 시 409를 반환한다.

## 2026-07-17 — 파티 계산 범위 결정

- 파티 기능의 첫 단계는 메인 캐릭터·지원 캐릭터 편성과 조건부 버프를 바탕으로 한 최종 스탯 합산으로 정했다. 결과 화면은 버프 제공자, 조건, 적용 여부를 근거로 표시한다.
- 전체 로테이션 DPS는 스킬·적·시간축 데이터와 검증 체계가 준비된 뒤 별도 단계로 진행한다. 현 단계에서 DPS 수치를 추정하지 않는다.

## 2026-07-17 — 파티 최종 스탯 계산 1차 구현

- 플래너에 지원 캐릭터 2명 편성을 추가하고, 메인 캐릭터와 편성된 지원 캐릭터의 조합에 따라 조건부 파티 버프를 적용 가능/미적용으로 판정한다.
- 결과는 버프 제공자, 버프 조건, 적용 여부와 미적용 사유를 함께 표시한다. 저장 API도 동일한 파티 규칙과 현재 데이터 릴리스의 캐릭터 존재 여부를 다시 검증한다.
- 개인 전용무기 버프는 파티 버프와 분리해 기존처럼 무기 조건으로 판정한다. 파티 규칙의 공식 지원 범위와 게임 데이터 릴리스 귀속은 후속 정합성 작업으로 관리한다.

## 2026-07-17 — 파티 계산 1차 검수 후 정합성 보강

- 파티 버프를 코드 상수에서 `party_buffs` 릴리스 데이터로 이전했다. 초안 복제와 발행 검증은 파티 버프를 함께 복제하고, 대상·제공 캐릭터 및 수치 형식을 검사한다.
- 새 환경에서 재현 가능한 공식 지원 파티 범위는 초기 시드가 포함한 장리–루파–브렌트로 확정했다. 히유키 관련 등급 계산식은 유지하되, 파티 버프 지원으로 문서화하지 않는다.
- 편성이 충족된 버프와 실제 최종 스탯에 반영된 버프를 구분해 표시하도록 안내를 추가했다. 파티 편성은 최대 2명의 지원 캐릭터를 허용한다.

## 2026-07-17 — 릴리스 기반 게임 데이터 관리 착수

- 패치 데이터를 행 단위로 즉시 덮어쓰지 않고, `draft` → `published` → `superseded` 상태를 갖는 `game_data_releases` 스냅샷으로 관리하기로 결정했다.
- 기존 명조 3.5 데이터를 첫 공개 릴리스로 이관하고, 캐릭터·무기·에코·세트·주옵션을 `release_id`로 귀속하는 마이그레이션을 추가했다.
- 플래너 조회와 서버 측 빌드 검증·저장이 현재 공개 릴리스만 사용하도록 전환했다. 새 저장 빌드는 `data_release_id`를 고정해 향후 패치와 무관하게 계산 기준을 추적한다.
- 다음 작업은 관리자 초안 복제·발행 API, 발행 전 도메인 검증, 관리자 화면의 draft/publish 흐름이다.

## 2026-07-17 — 관리자 draft/publish 워크플로 구현

- 관리자 화면을 릴리스 대시보드로 변경했다. 현재 공개 릴리스와 초안·과거 릴리스를 구분해 보여 주고, 공개본을 복제해 새 버전의 초안을 만든다.
- 기존 JSON 편집기는 유지하되 선택한 `draft`에만 저장하도록 제한했다. 공개·과거 릴리스는 읽기 전용이므로 운영 데이터가 행 단위로 즉시 변경되지 않는다.
- 초안 발행 전 서버가 데이터 최소 구성, 릴리스 버전·검증일 일관성, 에코 세트 소속 관계를 검사한다. 발행 확인 뒤 새 릴리스 공개, 이전 릴리스 보존, 현재 게임 릴리스 포인터 변경은 하나의 트랜잭션으로 수행한다.

## 2026-07-17 — 히유키·치사·루실라 응결 파티 및 에코 데이터 정비

- 히유키를 기준으로 치사·루실라 조합의 조건부 버프, 등급 목표치, 저장 빌드 재계산 로직을 추가했다. 응결 피해 보너스를 독립 스탯으로 계산·표시·검증한다.
- 히유키·치사·루실라, 전용 무기, `소리 없이 내려앉은 기도의 눈` 에코 세트와 개별 에코·세트 구성을 로컬 게임 데이터에 등록했다. 에코 세트 5세트 효과의 치명타 확률도 계산에 반영한다.
- 부옵션 입력을 자유 수치 입력에서 실제 에코 롤 수치 선택 방식으로 바꾸고, 에코당 최대 5개 및 유효 롤 값 검증을 적용했다.
- 에코 주옵션과 추천 문구의 스탯 레이블을 한글화했다. 응결 파티 관련 조건부 효과는 `서리 효과`라는 표기로 통일했다.
- PowerShell 입력 과정에서 발생한 한글 물음표 저장 문제를 복구했다. 캐릭터·무기·에코·세트 이름과 `소리 없이 내려앉은 기도의 눈` 5세트 조건을 정상 한글 및 JSON 객체 형태로 정정했다.
- `pnpm lint`, `pnpm test`, `pnpm build`를 통과했다.

## 2026-07-16 — 로컬 환경 설정 안정화
