# Buildex 개발 로그

## 2026-07-16 — 관리자 게임 데이터 운영

- `users.role`과 관리자 부트스트랩(`ADMIN_EMAIL`)을 추가했다. 일반 회원가입 계정은 기본 역할 `user`로 생성되며, 관리자 API는 서버에서 역할을 다시 검사한다.
- `/admin`과 `/api/admin/game-data`를 추가해 게임, 캐릭터, 무기, 에코, 에코 세트, 주옵션, 세트 구성을 등록·수정·삭제할 수 있게 했다.
- 입력은 Zod로 데이터 버전, 검증 날짜, 출처 URL 및 식별자를 검증한다. 기존 레코드를 불러와 JSON으로 편집할 수 있고, 같은 식별자로 저장하면 갱신된다.
- `pnpm lint`, `pnpm build`, `pnpm test`를 통과했다.

## 2026-07-16 — 기반 구축

- Next.js, PostgreSQL, Drizzle ORM, Auth.js Credentials 기반을 구성했다.
- 사용자, 인증 식별자, 게임 데이터, 빌드 프로필 스키마와 마이그레이션을 추가했다.
- 창리 기준 스탯 계산기와 단위 테스트를 추가했다.

## 2026-07-16 — 빌드 저장

- 로그인 사용자의 빌드 목록 조회, 생성, 삭제 API를 구현했다.
- 서버에서 빌드 입력을 검증하고 계산 결과와 버전을 함께 저장하도록 했다.

## 2026-07-16 — 데이터 선택 및 편집

- `/api/build-data`로 DB 기반 캐릭터·무기·에코 세트·주옵션 선택 데이터를 제공한다.
- 캐릭터에 맞는 무기 타입만 선택하도록 빌드 UI를 변경했다.
- 저장 빌드를 불러오고 수정(PUT)·삭제할 수 있게 했다.
- 문서 네 개를 UTF-8 한국어 원문으로 정규화했다.

## 2026-07-16 — MVP 에코 선택지 보완

- 1·3 코스트 에코와 세트 연결 데이터를 추가해 4-3-3-1-1 빌드 슬롯을 모두 선택할 수 있게 했다.
- 저장 입력에서 5개 슬롯, 슬롯별 코스트, 에코 선택 여부를 검증하도록 강화했다.
- 에코 데이터는 출처 URL과 스냅샷을 함께 기록해 다음 갱신의 기준으로 사용한다.

## 2026-07-16 — 에코 세트 효과 계산

- 에코 슬롯마다 세트를 선택해 2세트/5세트 조합을 표현할 수 있게 했다.
- 2세트 효과는 자동 적용하고, 조건이 있는 5세트 효과는 사용자 확인 후 클라이언트와 서버 계산에 동일하게 반영한다.

## 2026-07-16 — 빌드 참조 무결성 검증

- 서버가 무기 타입, 에코 코스트, 에코-세트 소속, 주옵션 가능 여부를 DB 기준으로 검증하도록 했다.
- 화염 발톱 세트의 1·3 코스트 선택지와 연결 데이터를 보충했다.

## 2026-07-16 — 에코별 부옵션 및 조건부 버프

- 부옵션을 에코별로 입력·저장·복원하도록 변경했다.
- 창리 전용무기 버프는 해당 무기 장착 시에만 표시하고 서버 계산에도 반영한다.

## 2026-07-16 — 등급 기반 부옵션 추천

- 창리 빌드는 다음 등급에 도달하기 위해 부족한 스탯을 결과 패널에서 바로 확인할 수 있게 했다.

## 2026-07-16 — 로컬 환경 설정 안정화

- `.env.local`에 로컬 데이터베이스 연결 정보와 고정 `NEXTAUTH_SECRET`, `NEXTAUTH_URL`을 추가했다.
- JWT 세션 복호화 오류를 방지하기 위해 NextAuth secret을 개발 서버 재시작 시에도 유지하도록 했다.
- `drizzle.config.ts`가 `.env.local`을 자동으로 읽도록 `dotenv`를 추가해, 마이그레이션 시 별도의 환경변수 명령이 필요 없게 했다.

## 2026-07-16 — 서버 주옵션 계산 데이터화

- 빌드 저장·수정 API가 주옵션 수치를 코드에 고정하지 않고, 검증에 사용한 `echo_main_stats` 게임 데이터에서 읽도록 통일했다.
- 서버 계산용 에코 스탯 소스를 공용 모듈로 분리하고, 데이터 값 변경이 계산에 반영되는 단위 테스트를 추가했다.

## 2026-07-16 — 저장 빌드 비교

- 현재 편집 중인 빌드와 저장 빌드를 선택해 공격력, 치명타, 공명 효율, 융해 피해 보너스 및 증감값을 한눈에 비교할 수 있게 했다.

## 2026-07-16 — 에코 부옵션 품질 검증

- 에코별 부옵션을 지원하는 항목으로 제한하고 중복 입력을 서버에서 거부하도록 검증을 강화했다.
- 각 에코의 유효 부옵션 개수와 치명타·속성 피해 중심의 비교용 품질 점수를 표시했다.

## 2026-07-16 — 등급 목표 행동 추천

- 다음 등급의 미달 스탯을 주옵션 교체 가능 여부와 연결해, 교체 후보 또는 우선 보강할 부옵션으로 안내하도록 확장했다.

## 2026-07-16 — 게임 데이터 운영 정보

- 빌드 데이터 API와 플래너에 현재 게임 데이터 버전, 검증 날짜 및 출처 링크를 노출했다.
- 패치 데이터 추가·검증·버전 갱신 절차를 문서화해 데이터 확장 시 계산 기준과 추적 정보를 유지하도록 했다.

## 2026-07-16 — 플래너 사용성 정리

- 캐릭터 변경 시 호환 무기 선택을 초기화해 표시와 저장 상태가 엇갈리지 않게 했다.
- 빌드 복제, 삭제 확인, 한국어 중심 문구 및 접근성 있는 빌드 이름 입력을 추가했다.

## 2026-07-16 — `/build` 화면 가독성 및 UI 개선

- 빌드 입력을 캐릭터·무기, 에코 세팅, 부옵션의 3단계 카드 구조로 재구성해 입력 순서를 명확히 했다.
- 결과 패널을 고정형 요약 영역으로 강화하고, 핵심 스탯·등급 목표·개선 추천·저장 기능을 시각적으로 분리했다.
- 저장 빌드 비교와 목록의 증감 표시를 정리하고, 폼 요소의 포커스 상태·모바일 레이아웃·한국어 안내 문구를 개선했다.
- `pnpm lint` 및 `pnpm build`로 타입 검사와 프로덕션 빌드를 검증했다.
