# Buildex 요구사항

> 버전 0.3 · 2026-07-16

## 목적

Buildex는 명조: 워더링 웨이브의 캐릭터 빌드를 입력하고 최종 스탯을 비교하는 웹 애플리케이션이다.

## MVP 범위

| ID | 요구사항 | 우선순위 | 상태 |
| --- | --- | --- | --- |
| FR-001 | 이메일 회원가입 | Must | 완료 |
| FR-002 | 로그인·로그아웃 | Must | 완료 |
| FR-003 | 소셜 로그인 | Must | 제외 |
| FR-004 | 캐릭터와 무기 선택 | Must | 완료 |
| FR-005 | 에코 세트와 주옵션 선택 | Must | 완료 |
| FR-006 | 부옵션을 포함한 스탯 계산 | Must | 완료 |
| FR-007 | 조건부 버프 반영 | Must | 완료 |
| FR-008 | 파티 시뮬레이터 | Could | MVP 제외 |
| FR-009 | 캐릭터별 등급 기준 | Must | 창리 기준 완료 |
| FR-010 | 빌드 페이지 접근 제어 | Must | 완료 |
| FR-011 | 빌드 저장·불러오기·수정·삭제 | Should | 완료 |

## 동작 원칙

- 빌드 계산과 저장은 로그인 사용자만 사용할 수 있다.
- 선택지는 DB의 캐릭터, 무기, 에코 세트, 에코 주옵션 데이터를 기준으로 표시한다.
- 무기는 캐릭터의 무기 타입과 일치하는 항목만 선택할 수 있다.
- 저장 시 서버가 계산 결과를 다시 산출해 저장한다. 클라이언트 계산 결과를 신뢰하지 않는다.
- 창리 외 캐릭터에는 아직 등급 기준을 적용하지 않는다.

## 비기능 요구사항

- 입력 데이터는 Zod로 검증한다.
- 사용자는 자신이 소유한 빌드만 조회·수정·삭제할 수 있다.
- 데이터 버전과 계산식 버전을 저장해 결과의 기준을 추적한다.
