# Buildex 요구사항 명세서

> 버전: 0.1 (MVP 초안) · 작성일: 2026-07-16

## 1. 우선순위 기준

- Must: MVP 출시를 위해 반드시 필요한 기능
- Should: MVP 품질을 높이지만 출시 후로 조정 가능한 기능
- Could: 사용성 향상을 위한 후보 기능

### 3.1 인수 조건

#### FR-001 회원가입

- 유효한 이메일과 비밀번호를 입력하면 계정이 생성된다.
- 이미 등록된 이메일이면 계정을 중복 생성하지 않고 안내한다.
- 비밀번호는 평문으로 저장하지 않는다.

#### FR-005 공개 정보 열람

- 로그인하지 않은 상태에서도 게임 선택, 캐릭터 상세, 장비 상세, 무기 상세 화면에 접근할 수 있다.
- 상세 페이지에는 해당 데이터의 게임과 기준 버전/갱신일을 표시할 수 있는 영역이 있다.

#### FR-006~009 회원 전용 기능

- 비회원은 기능 실행 시 로그인 또는 회원가입으로 이동한다.
- 로그인 성공 후 사용자가 시도했던 기능으로 복귀한다.
- 사용자 A의 저장 스펙·즐겨찾기·시뮬레이션 데이터는 사용자 B가 조회하거나 수정할 수 없다.

#### FR-009 파티 조합 시뮬레이션

- 파티 슬롯 수와 중복 캐릭터 허용 여부는 게임별 규칙으로 관리할 수 있다.
- 조합 결과에는 계산 기준, 입력 데이터 기준일, 결과 요약을 함께 표시한다.
- MVP에서는 결과 저장 여부를 선택 사항으로 둔다. 저장을 제공한다면 사용자 소유 데이터로 처리한다.

## 5. 범위와 제약

- 콘텐츠 조회는 공개, 개인 데이터 생성·보관·계산은 인증 필수라는 경계를 일관되게 적용한다.
- 게임 API 또는 외부 인증 API의 실제 도입은 제공사의 약관, 호출 한도, 개인정보 처리 정책을 확인한 뒤 진행한다.
- 계산식이 게임 업데이트에 따라 바뀔 수 있으므로, 결과는 항상 적용한 게임 버전과 계산식 버전을 함께 보관하는 것을 권장한다.

## 7. 초기 기술 기반

- Next.js App Router, TypeScript, React, Tailwind CSS
- PostgreSQL 16 Docker 구성과 Drizzle ORM 초기 마이그레이션
- Auth.js Credentials 기반 이메일/비밀번호 인증
- Zod 입력 검증, Vitest 단위 테스트, ESLint 및 프로덕션 빌드 검증
