# Buildex 요구사항

> 버전 0.4 · 2026-07-17

## 목적

Buildex는 명조: 워더링 웨이브의 캐릭터 빌드를 입력하고 최종 스탯을 비교하는 웹 애플리케이션이다.

## 동작 원칙

- 빌드 계산과 저장은 로그인 사용자만 사용할 수 있다.
- 선택지는 DB의 캐릭터, 무기, 에코 세트, 에코 주옵션 데이터를 기준으로 표시한다.
- 무기는 캐릭터의 무기 타입과 일치하는 항목만 선택할 수 있다.
- 저장 시 서버가 계산 결과를 다시 산출해 저장한다. 클라이언트 계산 결과를 신뢰하지 않는다.
- 에코 부옵션은 공격력·HP·방어력, 치명타, 공명 효율과 일반 공격·강공격·공명 스킬·공명 해방 피해 보너스만 지원하며, 서버가 실제 롤 값만 허용한다.
- 등급 기준과 전용 조건부 버프는 현재 창리·자니·히유키에 제공한다. 다른 캐릭터에는 최종 스탯은 계산하지만 등급과 개선 추천은 제공하지 않는다.
- 파티 계산은 메인 캐릭터와 최대 2명의 지원 캐릭터를 편성하고, 편성과 사용자 확인 조건을 만족한 버프만 최종 스탯에 합산한다. 1차 공식 지원 조합은 장리–루파–브렌트이며, 적용 가능·조건 미확인·최종 반영·편성 미충족 사유를 함께 표시한다.

## 운영 기능

- `ADMIN_EMAIL`과 일치하는 로그인 계정은 최초 인증 요청 시 관리자 역할로 승격된다. 일반 회원가입 계정은 항상 `user` 역할이다.
- 관리자는 `/admin`에서 현재 공개 릴리스를 복제해 초안을 만들고, 초안에서만 게임·캐릭터·무기·에코·세트·주옵션을 편집한다.
- 발행 전 서버가 데이터 구성과 버전·검증일 일관성을 검사한다. 발행은 새 릴리스 공개, 이전 릴리스 보존, 현재 릴리스 포인터 전환을 하나의 트랜잭션으로 처리한다.
- 모든 게임 데이터는 데이터 버전, 검증 날짜, 출처 URL을 함께 저장하며 관리자 API는 서버에서 역할을 다시 검사한다.

## 비기능 요구사항

- 입력 데이터는 Zod로 검증한다.
- 사용자는 자신이 소유한 빌드만 조회·수정·삭제할 수 있다.
- 데이터 릴리스와 서버가 적용한 계산식 버전을 저장해 결과의 기준을 추적한다. 과거 릴리스 기반 열람과 현재 릴리스로의 명시적 복제·재계산을 제공한다.

## 후속 기능

- 전체 로테이션 DPS 시뮬레이션은 별도 단계로 진행한다. 이를 위해 캐릭터 스킬 계수·시전 시간·자원·쿨다운, 적 방어력·저항, 버프 지속시간·중첩·스냅샷과 로테이션 타임라인 데이터를 축적하고 검증한다.
